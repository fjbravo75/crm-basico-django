# update_current_context.md

## Cuándo usar este workflow

Usa este workflow cuando haya cambiado la foto operativa vigente del proyecto.

Ejemplos típicos:

- cambia la fase actual
- cambia el bloque abierto
- cambia el objetivo inmediato
- cambia el estado general del repo o del proyecto
- cambia el siguiente paso exacto

---

## Pasos

1. Lee `AGENTS.md`.
2. Lee `project_memory/CURRENT_CONTEXT.md`.
3. Confirma que el cambio es real y afecta al presente operativo.
4. Reescribe `project_memory/CURRENT_CONTEXT.md` para reflejar el nuevo estado vigente.
5. No conviertas el archivo en histórico.
6. No dupliques decisiones que vivan mejor en `DECISIONS.md`.
7. Deja claro el siguiente paso exacto.

---

## No hagas esto

- no añadas histórico largo
- no registres microcambios irrelevantes
- no copies sesiones enteras
- no metas reglas del sistema documental
