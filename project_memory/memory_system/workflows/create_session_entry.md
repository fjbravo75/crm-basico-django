# create_session_entry.md

## Cuándo usar este workflow

Usa este workflow cuando una sesión o bloque tenga entidad histórica propia y convenga dejar un cierre útil para continuidad futura.

---

## Pasos

1. Lee `AGENTS.md`.
2. Lee `project_memory/memory_system/NAMING_CONVENTIONS.md`.
3. Usa `project_memory/memory_system/templates/session_entry_template.md`.
4. Crea el archivo dentro de `project_memory/sessions/` con el nombre correcto.
5. Completa el front matter YAML mínimo.
6. Redacta el cuerpo con:
   - estado al cierre
   - trabajo realizado
   - decisiones fijadas o confirmadas
   - pendiente inmediato
   - siguiente paso exacto
7. No conviertas la sesión en diario largo.
8. Actualiza `project_memory/sessions/sessions_index.md`.

---

## No hagas esto

- no crees sesión por microavance irrelevante
- no dupliques entero `CURRENT_CONTEXT.md`
- no uses la sesión como lugar para teoría del sistema
