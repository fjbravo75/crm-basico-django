# update_decisions.md

## Cuándo usar este workflow

Usa este workflow cuando nazca, cambie o quede fijada una decisión activa relevante que siga condicionando trabajo futuro.

---

## Pasos

1. Lee `AGENTS.md`.
2. Lee `project_memory/DECISIONS.md`.
3. Confirma que la decisión es realmente vigente y no una hipótesis.
4. Actualiza `project_memory/DECISIONS.md`.
5. Si la nueva decisión sustituye una anterior, no la dejes mezclada sin más.
6. Si corresponde, usa también `archive_decision.md`.
7. Mantén `DECISIONS.md` como lista viva de criterios vigentes.

---

## No hagas esto

- no conviertas `DECISIONS.md` en histórico
- no metas dudas abiertas como decisiones cerradas
- no repitas contexto que ya vive mejor en `CURRENT_CONTEXT.md`
