# WRITING_RULES.md

## Propósito de este archivo

Este archivo fija las reglas de redacción para todos los documentos del sistema de memoria local.

---

## Reglas generales de redacción

- Escribe con estilo técnico, claro, breve y verificable.
- Evita narrativa innecesaria.
- Evita contexto ornamental.
- Prioriza hechos, decisiones, pendientes y siguiente paso.
- No uses frases vagas.
- No uses cierres vacíos.
- No maquilles lagunas con texto genérico.
- No uses el documento para “dejarlo bonito”.
- Escribe para retomar trabajo, no para impresionar.

---

## Reglas de precisión

- Distingue claramente entre hecho, decisión, pendiente y siguiente paso.
- No mezcles categorías dentro de frases ambiguas.
- Si algo no se sabe, dilo con claridad.
- Si algo no está validado, no lo presentes como validado.

---

## Reglas de cierre útil

Siempre que aplique, deja explícito:

- qué estado queda al cierre
- qué se hizo realmente
- qué quedó fijado
- qué queda pendiente
- cuál es el siguiente paso exacto

---

## Reglas de brevedad útil

- Usa solo el nivel de detalle necesario para continuidad real.
- Evita repetir contexto que ya vive mejor en otra capa.
- No conviertas una entrada útil en un bloque pesado de lectura.

---

## Reglas por tipo de pieza

### Sesiones

Escribe como cierre operativo de un bloque.

### Decisiones archivadas

Escribe como trazabilidad breve de algo que dejó de estar vigente.

### Índices

Escribe como instrumento de localización, no como resumen ampliado.

### `CURRENT_CONTEXT.md`

Escribe como foto presente del proyecto.

### `DECISIONS.md`

Escribe como lista viva de criterios vigentes.

---

## Regla final

Si un texto no ayuda a retomar, decidir o verificar, probablemente sobra.
