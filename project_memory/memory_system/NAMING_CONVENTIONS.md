# NAMING_CONVENTIONS.md

## Propósito de este archivo

Este archivo fija las convenciones de nomenclatura del sistema documental local.

---

## Convenciones para sesiones

### Nombre de archivo

Formato:

`YYYY-MM-DD__slug-del-bloque.md`

Ejemplo:

`2026-03-19__memory-system-selection-rules.md`

### ID interno

Formato:

`session-YYYY-MM-DD-XX`

Ejemplo:

`session-2026-03-19-01`

---

## Convenciones para decisiones archivadas

### Nombre de archivo

Formato:

`YYYY-MM-DD__topic__slug.md`

Ejemplo:

`2026-03-19__architecture__split-memory-layers.md`

### ID interno

Formato:

`archived-decision-YYYY-MM-DD-XX`

Ejemplo:

`archived-decision-2026-03-19-01`

---

## Convenciones para índices

### Nombres de archivo fijos

- `sessions_index.md`
- `decisions_archive_index.md`

### IDs internos

- `sessions-index`
- `decisions-archive-index`

---

## Convenciones de slug

- usa minúsculas
- usa guiones medios
- evita espacios
- evita caracteres decorativos
- el slug debe describir el bloque de forma breve y reconocible

---

## Convenciones generales

- no improvises nuevos formatos si ya existe una convención fijada
- mantén consistencia total entre nombre de archivo e id
- si dudas entre dos nombres, usa el más claro y más simple
